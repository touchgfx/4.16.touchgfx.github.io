Getting Started

설치 섹션에 설명되어 있듯이, TouchGFX 사용을 시작하려면 먼저 최신 버전의 TouchGFX를 설치 해야 합니다.

설치가 완료되고 나면 첫 번째 TouchGFX 프로젝트를 시작할 준비가 됩니다. 가장 기본적인 TouchGFX 개념을 숙지하려면 본 설명서의 몇 가지 주요 섹션들을 꼭 읽어보시기 바랍니다. 가장 기본적인 섹션은 개발 소개(Development Introduction) 로, 완벽한 TouchGFX 프로젝트에 필요한 소프트웨어 및 하드웨어 구성 요소를 비롯해 이들을 생성하기 위한 활동 및 도구들이 소개되어 있습니다. 이 외에도 사전 제작된 구성 요소를 사용하여 프로토타이핑을 빠르게 시작하는 방법이 나와 있습니다.

TouchGFX Designer Quick Start​

Application Template + UI Template​

TouchGFX를 실험적으로 사용해 보고 싶은 분들은 UI 프로젝트 예시를 이용해 STM32 평가 키트에서 실행하는 방식으로 TouchGFX Designer를 시작할 수 있습니다.

Application template과 UI template을 통해 간단하게 프로젝트 생성

TouchGFX Designer의 시작 창에서 다양한 UI 예시 가운데 원하는 것을 선택하여 시작할 수 있습니다. 더 나아가 이를 다양한 STM32 평가 키트에 대한 보드별 코드에 통합하여 임베디드 하드웨어에서 예시를 즉시 실행해 볼 수 있습니다.

STM32 평가 키트가 없는 경우에는 PC에서 간단하게 TouchGFX 애플리케이션을 실행할 수 있습니다. 초반 단계들에 대한 지침은 튜토리얼 챕터에서 찾아볼 수 있습니다. 여기에는 프로젝트를 설치 및 실행하여 다소 복잡한 UI 애플리케이션을 구현하는 방법이 나와 있습니다.

Online Applications​

TouchGFX Designer에는 UI template에 application template을 결합할 수 있는 기능 외에도 특정 STM32 평가 키트에 대한 특별 데모가 몇 가지 포함되어 있습니다. 이들 데모는 Online Applications 탭 아래에 있습니다.

다음 단계​

여기서 설명서를 쭉 살펴보면서 자신에게 해당하는 챕터를 읽어보시기 바랍니다. 이렇게 설명서의 개요를 읽고 자세한 정보의 위치를 알 수 있으며, TouchGFX 주요 개념들을 숙지할 수 있습니다.

주요 챕터들은 다음과 같습니다:

  • Basic Concepts: 이 챕터는 설명서의 나머지 부분들에 대한 배경 지식 역할을 합니다. 여기서는 이후에 렌더링 되는 모든 주요 그래픽 개념들을 소개합니다. 여기서는 이후에 렌더링 되는 모든 주요 그래픽 개념들을 소개합니다. 현재 지식과 수행하게 될 TouchGFX 개발 활동에 따라 몇몇 섹션들은 건너 뛰고 나중에 세부적인 내용이 궁금할 때 참조할 수 있습니다.
  • Development: 본 설명서의 메인 챕터입니다. TouchGFX 프로젝트 구조를 비롯해 전체 프로젝트 개발 주기에 포함되어 있는 워크플로와 도구들이 설명되어 있습니다. 워크플로의 각 단계는 자체 섹션을 통해 자세히 설명되어 있습니다. 여기서 TouchGFX 프레임워크의 모든 기능과 구성 요소에 대한 설명을 포함해 하드웨어에서 애플리케이션을 실행하는 방법과 UI 개발 방법을 이해하는 데 필요한 모든 정보를 찾을 수 있습니다.
